# 模型格式說明 - 如何保留 Tinkercad 紋理

## 問題說明

STL 格式**不支援紋理貼圖**，只支援頂點顏色。當你從 Tinkercad 轉存 STL 時，所有的表面紋理、材質和顏色資訊都會丟失。

## 解決方案

### ✅ 推薦方案：使用 GLTF/GLB 格式（最佳選擇）

**GLTF/GLB 格式的優點：**
- ✅ 完整保留紋理和材質
- ✅ 檔案大小適中
- ✅ 載入速度快
- ✅ 現代 3D 網頁標準格式

**從 Tinkercad 匯出 GLTF：**
1. 在 Tinkercad 中開啟你的模型
2. 點擊右上角的「匯出」按鈕
3. 選擇 **「GLTF」** 格式
4. 下載檔案（會是 `.glb` 或 `.gltf` 格式）

**使用方式：**
1. 將下載的 `.glb` 或 `.gltf` 檔案放到專案目錄
2. 重新命名為 `JHSNU_MAP.glb`（或 `.gltf`）
3. 在瀏覽器控制台執行：
   ```javascript
   map3D.setModelPath('JHSNU_MAP.glb');
   ```
4. 頁面會自動重新載入新模型

---

### 方案 2：使用 OBJ 格式

**OBJ 格式的優點：**
- ✅ 支援紋理貼圖
- ✅ 廣泛支援
- ⚠️ 需要額外的 MTL 材質檔

**從 Tinkercad 匯出 OBJ：**
1. 在 Tinkercad 中開啟你的模型
2. 點擊「匯出」→ 選擇 **「OBJ」**
3. 下載檔案（會包含 `.obj` 和 `.mtl` 檔案）

**使用方式：**
1. 將 `.obj` 和 `.mtl` 檔案都放到專案目錄
2. 重新命名 `.obj` 為 `JHSNU_MAP.obj`
3. 確保 `.mtl` 檔案也在同一目錄
4. 在瀏覽器控制台執行：
   ```javascript
   map3D.setModelPath('JHSNU_MAP.obj');
   ```

---

### 方案 3：繼續使用 STL（不推薦，會失去紋理）

如果你不需要紋理，可以繼續使用 STL 格式。但**紋理資訊會完全丟失**。

---

## 快速切換模型格式

如果你想在網站上切換不同的模型檔案，可以在瀏覽器控制台（F12）執行：

```javascript
// 載入 GLTF 格式（推薦）
map3D.setModelPath('JHSNU_MAP.glb');

// 載入 OBJ 格式
map3D.setModelPath('JHSNU_MAP.obj');

// 載入 STL 格式
map3D.setModelPath('JHSNU_MAP.stl');
```

系統會記住你的選擇，下次開啟會自動載入你選擇的格式。

---

## 注意事項

1. **檔案命名**：確保檔案名稱與程式碼中指定的名稱一致
2. **檔案位置**：模型檔案必須與 `index.html` 在同一目錄
3. **本地伺服器**：必須使用本地伺服器開啟（不能直接用 `file://` 開啟）
4. **材質檔案**：如果使用 OBJ 格式，確保 `.mtl` 材質檔也在同一目錄

---

## 技術說明

本專案現在支援以下格式：
- **STL** - 僅支援頂點顏色，不支援紋理
- **OBJ** - 支援紋理貼圖（需要 MTL 材質檔）
- **GLTF/GLB** - 完整支援紋理、材質、動畫（推薦）

所有格式都會自動：
- 調整到正確的座標系（Y 軸朝上）
- 自動縮放到合適大小
- 設定陰影和光照
